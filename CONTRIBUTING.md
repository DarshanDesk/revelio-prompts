# Contributing to Revelio Prompts

Thank you for your interest in contributing to this prompt collection! This guide will help you add your own prompts effectively.

## üìã Prompt Structure

Each prompt should be a markdown file following this template:

```markdown
# Prompt Title

## Category
[Category Name]

## Description
Brief description of what this prompt does and when to use it.

## Prompt
```
[Your prompt text here with clear placeholders in [BRACKETS]]
```

## Usage Tips
- Practical tip 1
- Practical tip 2
- Additional guidance

## Examples
Brief description of use cases or example scenarios.
```

## üìÅ Directory Structure

Add your prompts to the appropriate category folder:

- `prompts/development/` - Programming, debugging, code review
- `prompts/writing/` - Content creation, editing, copywriting
- `prompts/analysis/` - Data analysis, research, insights
- `prompts/creative/` - Brainstorming, storytelling, creative work
- `prompts/productivity/` - Task management, planning, organization

If your prompt doesn't fit existing categories, suggest a new one!

## ‚úÖ Guidelines

1. **Clear Placeholders**: Use `[DESCRIPTIVE BRACKETS]` for user inputs
2. **Actionable**: Prompts should produce useful, actionable outputs
3. **Well-Tested**: Test your prompt before submitting
4. **Concise**: Keep descriptions brief but informative
5. **Practical**: Include usage tips from real experience
6. **Original**: Share prompts you've created or significantly adapted

## üöÄ Submitting Prompts

1. Fork the repository
2. Create a new branch (`git checkout -b add-my-prompt`)
3. Add your prompt file to the appropriate category
4. Follow the naming convention: `lowercase-with-hyphens.md`
5. Commit your changes (`git commit -m 'Add [prompt name]'`)
6. Push to your branch (`git push origin add-my-prompt`)
7. Open a Pull Request

## üí° Prompt Quality Checklist

Before submitting, ensure your prompt:

- [ ] Has a clear, descriptive title
- [ ] Includes all required sections (Category, Description, Prompt, Usage Tips, Examples)
- [ ] Uses consistent formatting
- [ ] Has proper markdown syntax
- [ ] Contains clear placeholders for user input
- [ ] Provides practical usage tips
- [ ] Includes realistic examples or use cases
